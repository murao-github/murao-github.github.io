javascript(jquery)を書く場所とLaravelMixの使い方についての備忘録

 
  • PHP
  • Laravel
  • 2019 01 24

LaravelでjQueryの関数を呼び出そうとしたとき、どこにコードを書いてどう呼び出せば良いのか分からなかったので、備忘録として記事を書いておこうと思います。

本記事に書かれている手順が必ずしも正しいというわけではありません。また、バージョンの差異などにより本記事の内容が有効でない場合がございます。ご了承ください。

ディレクトリ構造 5.7 Laravel

ディレクトリの配置場所なんかは、Laravelの設計思想上「composerが動いてたらどこでもOK!」というふわっとしたルールになっています。そのため、本記事に書かれている配置が気に食わないと感じる方もいると思います。その場合は自分のお好きなディレクトリを選択してください。

本記事の手順を実行するには、npmとnodeが必須となりますので、予めご自身の開発環境でこれらが利用できるように準備しておいてください。

今回の手順は純粋なjavascript(vanilla.js)でも、jQueryでも大体同じ手順で動作させらます。

開発環境

  • OS Windows 7
  • Apache 2.4.35
  • MySQL 15.1
  • php 7.2.11
  • Laravel 5.7
  • node.js 10.15.0
  • npm 6.4.1

手順

ソースコードの配置場所を決める

書いたjsコードは、resources/js配下に適当なフォルダを作って、その中に配置していきます。

Directory Structure – Laravel – The PHP Framework For Web Artisans

初めはpublic/js内に配置していましたが、英語版公式のドキュメントを見ると「コンパイル前の生のjavascript、LESS、SASSなんかはresourcesに入れてね！」って書いてあるので、それに従いたいと思います。

resources
 ├─/js
 │  │
 │  ├─/asset
 │  │  ├─test_jquery.js
 │  │  └─test_vanilla.js
 │  │
 │  ├─/components
 │  │  └─ExampleComponent.vue
 │  ├─bootstrap.js
 ~  └─app.js

resourcesフォルダ内に新しくassetフォルダを作成し、その中にそれぞれのフォーマットのソースコードを配置します。（これらの異なったフォーマットの.jsコードをひとつのプロジェクト内で書く人はあまりいないかもしれませんが…）

今回は、例としてtest_jquery.jsとtest_vanilla.jsを作成して説明を進めていきます。

javascriptのソースを書く

今回は説明用に、以下のような処理を記述しました。

// test_vanilla.js
function test_vanilla() {
    alert("Hello Vanilla Javascript!!");
}

// test_jquery.js
$(function () {
    $('#btn').on('click', function() {
        alert("Hello jQuery!!");
    });
});

書いたソースをwebpack.mix.jsに登録する

ここからは、webpack、Laravel Mixの設定についてです。

webpackやLaravel Mixについての詳細な説明は行うと長ったらしくなるので、とりあえずjsコードを動かしてみるところまでを目指します。 「詳しい説明を読みたい！」という方は以下の公式ドキュメントをどうぞ。

アセットのコンパイル(Mix) 5.7 Laravel

JavaScriptとCSSスカフォールド 5.7 Laravel

Laravelのプロジェクトフォルダ内にwebpack.mix.jsというファイルが存在すると思います。この中に新しく書いた.jsファイルを登録していきます。

// webpack.mix.js (はじめから書いてある)
mix.js('resources/js/app.js', 'public/js')
   .sass('resources/sass/app.scss', 'public/css');

上の処理に続けて、新しく追加したソースコードをビルドするよう、メソッドをチェーンしていきます。

// webpack.mix.js (新しく作った jQuery, vanilla のソースコードのパスを追加)
mix.js([
        'resources/js/app.js',
        'resources/js/assets/test_jquery.js',
    ], 'public/js/app.js')
    .scripts('resources/js/assets/test_vanilla.js', 'public/js/test_vanilla.js')
    .sass('resources/sass/app.scss', 'public/css');

jQueryの場合

jQueryのソースコード（test_jquery.js）は、初めから記述してあるmix.js()の第1引数を配列にして追加します。こうすることで、resources/js/app.jsとresources/js/app.jsをpublic/js/app.jsという1つのファイルにまとめる（ビルド）ことができます。

javascript(vanilla.js)の場合

javascript(vanilla.js)のソースコード（test_vanilla.js）は、新しく追加したmix.scripts()メソッドで登録しましょう。

今回は使いませんが、mix.scripts()と類似したメソッドとしてmix.babel()が用意されています。これは、ES6やECMAScript7などの新しい規格を基に書かれた.jsコードを、一般的なブラウザが理解できるES2015(ES6)の形式に変換（トランスパイル – transpile とか言うらしい）してくれるメソッドらしいです。便利やん…。

npm run devでソースをコンパイルする

次に、コンソールを開きます。いつもphp artisan ~コマンドを実行しているコンソールで結構です。

コンソールが開けたら以下のコマンドを実行してください。

npm run dev

実行するとコンパイルが開始されますので、しばらく待ちましょう。 終了すると、public/jsフォルダ内に新しくtest_vanilla.jsができているはずです。app.jsの最後行付近にも先ほど記述したtest_jquery.jsのコードが追加されていると思います。

おめでとうございます！あとはviewの中からこのファイルを読み込んでやるだけです。

ちなみに、watchコマンドを利用することで、ファイルの内容が変更されたことを検知し、自動的に再コンパイルを行うことができます。

npm run watch

特定の環境下ではwatchコマンドを利用していても再コンパイルが行われないことがあります。そういうときはwatch-pollコマンドを利用しましょう。

npm run watch-poll

view内でassetを読み込む

最後に、viewの中で先ほどコンパイルしたassetを読み込みましょう。

Laravel 5.7にデフォルトで存在するapp.blade.phpを利用している場合、app.jsは初めから読み込まれているため、test_vanilla.jsのみを読み込んでやります。

今回はapp.blade.phpではなく、以下のようなbladeテンプレートを使って説明していきます。


<!-- test.blade.php -->
<!doctype html>
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
 
        <!-- CSRF Token -->
        <meta name="csrf-token" content="{{ csrf_token() }}">
 
        <title>{{ config('app.name', 'Laravel') }}</title>
 
        <!-- Scripts -->
        <script src="{{ asset('js/app.js') }}" defer></script>
 
        <!-- Fonts -->
        <link rel="dns-prefetch" href="//fonts.gstatic.com">
        <link href="https://fonts.googleapis.com/css?family=Nunito" rel="stylesheet" type="text/css">
 
        <!-- Styles -->
        <link href="{{ asset('css/app.css') }}" rel="stylesheet">
    </head>
 
    <body>
        <div id="app" class="p-3">
            <p>Vanilla.js, jQuery のテスト</p>
            <button onclick="test_vanilla()">test_vanilla</button>
            <button id="btn">test_jquery</button>
        </div>
 
        <!-- vanilla.js はここで読み込む-->
        <script src="{{ asset('js/test_vanilla.js') }}"></script>
    </body>
</html>

app.jsは14行目で、test_vanilla.jsは32行目のbodyタグが終わる直前で読み込んでいます。app.blade.phpなどのテンプレートを継承して利用する場合は、子テンプレートからscriptタグを挿入できるように工夫するとよいでしょう。

routeを通して、jsが動作するか試してみる

# web.php
Route::get('test', function () {
    return view('test');
});

通したrouteにアクセスしてみましょう。

こんな風にbuttonが2つ並んで表示されると思います。

左のボタンを押すと『Hello Vanilla Javascript!!』と表示されます。

右のボタンを押すと『Hello jQuery!!』と表示されます。
